# Linux Logs and Journal

## What are Logs?

Logs are records of events generated by the Linux kernel, services, and applications.

They help administrators monitor the system, troubleshoot problems, detect errors, and investigate security incidents.

Examples of logged events:

- User login
- Service started or stopped
- Hardware detected
- Network connection
- System errors

---

## What is journald?

journald is a system service provided by systemd.

Its job is to:

- Collect logs from the kernel.
- Collect logs from services.
- Collect logs from applications.
- Store all logs in the system journal.

---

## What is the Journal?

The Journal is the storage location where journald keeps all collected logs.

It acts as a central database for system logs.

---

## What is journalctl?

journalctl is a command-line tool used to read and display logs stored in the Journal.

It allows administrators to search, filter, and monitor system logs.

---

## Logging Architecture

```text
Kernel
      \
Services \
Programs  \
           ↓
       journald
           ↓
        Journal
           ↓
      journalctl
           ↓
          User
```

---

## Common journalctl Commands

### Show all logs

```bash
journalctl
```

Displays all available logs.

---

### Show the last 50 log entries

```bash
journalctl -n 50
```

Displays the most recent 50 log messages.

---

### Follow logs in real time

```bash
journalctl -f
```

Shows new log entries as they are generated.

---

### Show logs for a specific service

```bash
journalctl -u service_name
```

Example:

```bash
journalctl -u ssh
```

Displays logs only for the SSH service.

---

### Show logs since the last boot

```bash
journalctl -b
```

Displays logs generated after the current system boot.

---

### Show only Kernel logs

```bash
journalctl -k
```

Displays log messages generated by the Linux kernel.

---

## Summary

- Logs record system events.
- journald collects and stores logs.
- The Journal is the central storage for logs.
- journalctl is used to read and analyze logs.
- System logs are essential for troubleshooting, monitoring, and security analysis.
